Satisfy "keep a Decisions section" via an index table (§12), leaving full decisions inline in context

Final design pass, 2026-07-01 · docs/planning/02-design.md · orchestrator judgment call

What was decided

The final doc's required Decisions section is a 32-row index table (ID → section → one-line summary, including retired D24), added as §12. The full decision text with rationale stays inline in each topical section, exactly as reviewed.

Alternatives considered

A. Move all decision bodies into one giant Decisions section Would strip each decision of its surrounding context (D26 makes sense next to the delivery-flow prose, not in a flat list), produce a large diff against the approved draft, and risk introducing errors during a polish-only pass. Rejected.
B. Duplicate full decision text (inline + Decisions section) Two copies of 31 decisions guarantees drift the first time one is edited. Rejected.
C. Index table pointing at inline decisions (chosen) One-screen scan of everything binding, zero duplication, near-zero diff risk against the approved content. This also gives implementers a checklist: each row is a thing the implementation phase must honor.

Inline (§6, unchanged from draft):
  "Decision D26: delivery is tracked with a delivery ledger, run_chunk (...)"

  | D26 | §6 | Per-chunk delivery ledger (run_chunk) as the fine-grained idempotency key |
