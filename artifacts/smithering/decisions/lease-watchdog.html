E2 — Run leases, heartbeats, and fenced writes replace the fixed-timeout watchdog

Context. REQ-9 requires legitimate tasks > 20 minutes; AC-8.1/9.4 require crash recovery. The v1 watchdog ("re-invoke anything running > 30 min") cannot tell a live 45-minute task from a dead one → duplicate live executors racing the delivery path.

Decision

Each executor start is an attempt that atomically acquires a lease: lease_owner (uuid, doubles as fencing token), lease_expires_at = now()+5min, heartbeat_at, attempt_count++. Heartbeat every 30 s extends the lease (fenced; a failed heartbeat = lease lost = abort immediately). The tick's watchdog re-invokes only runs with lease_expires_at < now() and attempt_count < 5; then finalizes failed with last_error. Every durable executor write carries WHERE lease_owner=$attempt.

Key property: lease TTL (5 min) is decoupled from task duration — a healthy 40-minute task heartbeats ~80× and is never taken over; a crashed one is recovered within ≤5 min.

Alternatives

Lease + heartbeat + fencing (chosen)Standard (graphile-worker locked_at/locked_by, k8s leases); distinguishes slow from dead; fencing makes stale writers harmless.
Fixed running-timeout (v1)False takeover of slow tasks; no mutual exclusion → duplicate senders.
Postgres advisory locks held by executorServerless/Sandbox connections via pooler make session-scoped locks unreliable (connection reuse/drop ≠ process death signal).
External coordination (Redis, workflow engine)Violates REQ-8 single-Postgres-store guarantee.

Example timeline (crash at t=7 min of a 40-min task)

t=0     attempt#1 acquires lease (uuid A), heartbeats 30s
t=7m    process dies; lease expires t=12m
t=12m   tick watchdog: lease_expires_at < now → invoke attempt#2 (uuid B)
t=12m+  attempt#2 re-reads run/run_chunk ledger, resumes at first unsent chunk
(late)  a revived attempt#1 write: ...WHERE lease_owner='A' → 0 rows → exits

Tests: concurrent-acquire property test (exactly one winner); slow-but-alive NOT taken over; kill + clock-advance recovery; attempt exhaustion → failed; fenced stale write hits zero rows. UI stall display driven by heartbeat_at.
