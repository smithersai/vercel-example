E3/E4 — Postgres-backed Telegram fake + fixture-mode Summarizer

Context. AC-10.2/10.3 require the same e2e suite green locally and against each PR's Vercel Preview. In Preview, webhook/tick/executor/UI run in separate isolates — the v1 in-memory Telegram fake's state is invisible across them. And C8's Anthropic dependency would make every PR lane paid, flaky, and non-deterministic.

Decision E3 — fake TelegramPort writes to Postgres

The fake adapter records every call (method, chat_id, payload, generated message_id) in a telegram_outbox table in the same per-Preview Neon branch (D29). The e2e suite asserts via a test-only authed route /api/test/outbox (enabled by E2E_TEST_ROUTES=1, D18 bearer secret, excluded from production builds). Identical assertion surface locally.

Decision E4 — SummarizerPort with fixture default

SUMMARIZER_MODE=fixture (default in all PR lanes) replays recorded outputs keyed by a hash of the canonicalized input; missing fixture = loud failure. anthropic mode (real API + schema validation + one repair retry) runs in a nightly/opt-in live-eval lane, mirroring the D30 Telegram smoke pattern. Both modes validate against the same SummarySchema, so shape bugs fail identically. Fixtures recorded via pnpm fixtures:record and reviewed in PRs.

Alternatives

Postgres outbox + fixture LLM (chosen)Durable across isolates; branch-isolated per PR for free; zero new infra; deterministic, secret-free PR lanes.
In-memory fake (v1)State lost across Preview isolates — the review's objection; suite could never pass in Preview.
Separate mock-Telegram HTTP serviceA second deployment to operate; violates the "nothing to operate" template spirit and complicates Preview wiring.
Live Anthropic in every PR laneNon-deterministic, paid, secret-dependent; blocks fork PRs; AC-3.2 assertions flake.
Assert only via UI/DB (no delivery surface)Cannot verify chunk order/content/protocol steps — exactly what AC-5.1/5.3 tests need to observe.

Example e2e assertion (same code, both environments)

await seedMessages(chat, fixture.messages)
await post('/api/cron/tick')                 // claim + execute
const out = await get('/api/test/outbox?chat=' + chat.id)
expect(out.filter(m => m.method === 'editMessageText')
          .map(m => m.payload.text).join(''))
  .toEqual(expectedSummaryChunks.join(''))   // exactly once, in order
