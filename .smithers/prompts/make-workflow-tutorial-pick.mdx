---

**Your agent will now ask you:** which one would you like to build?

Tell the agent:
1. **Which workflow** you want (by `name` or rank)
2. Any **modifications** — rename it, adjust the goal, expand or narrow scope
3. Any **additional context** that would help build it better (e.g. "use the GitHub API", "output to Slack", "integrate with our Linear board")

Once you reply, Smithers will run `create-workflow` to:
- Design the workflow's JSX task graph
- Scaffold `.smithers/workflows/{name}.tsx` + `.smithers/ui/{name}.tsx`
- Verify the graph renders
- Launch it, monitor it, and self-improve until it works

**Note:** `create-workflow` itself is a Smithers workflow — you're about to watch a workflow
build another workflow. That's the whole idea.
