# Verification Backpressure Matrix — v1 (2026-07-01)

Upstream: `docs/planning/01-prd.md` (REQ/AC), `docs/planning/03-eng.md` (E-decisions,
§14 lane matrix, §16 probes). One gate row per acceptance criterion. Every blocking
gate is an executable check that can FAIL; "the agent said it was done" is never
evidence. The implementation workflow wires every **blocking** gate into merge checks;
nightly-lane blocking gates block *release/acceptance*, not per-PR merge, and are
marked `(nightly)` in checkedBy.

Recorded decisions in this matrix:

- **BP-1**: Where a property/unit test fully proves a criterion (e.g. AC-2.1 lineage,
  AC-2.2 DST), it is the gate; the e2e suite covers the same behavior incidentally but
  is not the named prover — cheapest method that actually proves wins.
- **BP-2**: AC-3.3 (no fabrication) can only be proven by the grounding eval over live
  LLM output; it runs nightly and is blocking for acceptance. The per-PR proxy
  (SummarySchema validation in fixture mode) is wired as part of AC-3.2's gate.
- **BP-3**: AC-5.3 has a dual gate: the crash-cut/concurrency test battery (blocking,
  per-PR) AND human acceptance of the A13 placeholder-residual amendment (blocking
  approval; §5 implementation may not merge before it).
- **BP-4**: AC-9.5 is informational by its own text ("supporting evidence only").
- **BP-5**: Red-before-green evidence (link to the committed failing run) is required
  for every gate marked `RBG` in evidenceRequired, per eng §18.
- **BP-6**: Fresh-eyes README gates (AC-8.3, AC-11.1) are agent reviews executed by an
  agent with no repo context following only the README; the agent's transcript plus the
  green e2e run against the resulting deployment are the evidence — the review alone is
  not.

## Merge-check wiring summary

- **Per-PR required checks (blocking, merge)**: all `unit_test` / `integration_test` /
  per-PR `e2e_test` rows above — concretely the jobs: unit+integration w/ 100%
  coverage (REQ-10.1), e2e local (REQ-10.2), e2e Preview (REQ-10.3), long-task PR
  variant (REQ-9.3), smoke skip-behavior leg (REQ-10.4).
- **Nightly blocking (release/acceptance, not per-PR merge)**: REQ-3.3 grounding eval,
  REQ-9.2 >20-minute proof.
- **Approval gates**: A13 placeholder-residual PRD amendment (folded into REQ-5.3) —
  §5 implementation may not begin before acceptance.
- **Human/agent walkthrough gates (release)**: REQ-8.3, REQ-11.1, REQ-10.5.
- **Informational**: REQ-9.5 only.
